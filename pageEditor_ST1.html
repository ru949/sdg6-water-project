3. Shared Template & Global CSS

As Student 1, I was responsible for creating the shared HTML template and the global CSS file that all four team members used across the entire website. This was one of the most important tasks in the project because any errors I made would affect everyone's pages.

HTML Structure

The shared template uses the following key HTML elements:

  • <header> — Contains the site logo and navigation bar
  • <nav> — Wraps the navigation links list
  • <ul> / <li> / <a> — Navigation links to all main pages
  • <main> — Placeholder where each student adds their own page content
  • <footer> — Contains the brand description, navigation column, team column, student bar with Page Editor and Validation links, and the copyright line

CSS Techniques

The global CSS file (style.css) defines the visual identity for the entire website. I used CSS custom properties (variables) to define the colour palette once and reuse it everywhere. Key layout techniques include:

  • Float-based layout — Used for the footer columns, card grids, and multi-column sections, keeping the code simple and compatible
  • Overflow: hidden with clearfix — Applied after float containers to prevent layout collapse
  • Position: sticky — Used on the header to keep navigation visible while scrolling
  • CSS variables — Defined in :root so colours, fonts, and spacing are consistent and easy to update
  • Box-shadow — Applied to cards for subtle depth without complex effects
  • Media queries — Used at 860px and 560px breakpoints to make the site responsive on tablet and mobile

Design Rationale

I chose a deep navy (#1b2e4d) and ocean blue (#1a9fd4) colour palette because it directly reflects the theme of water and reinforces the seriousness of the SDG 6 subject matter. The Lato font was chosen for its clean, professional appearance and excellent readability at small sizes. I deliberately kept the CSS techniques basic — using float layouts rather than CSS Grid or Flexbox — to ensure the code is understandable and consistent with the level expected at this stage of the course.

4. Home Page (home.html)

The Home Page is the central entry point to the Project Blue website. It introduces the SDG 6 crisis, presents three mission cards, displays a statistics bar, and provides an interactive gallery of tiles linking to all four student content pages.

HTML Structure

  • <section class="hero"> — Full-height opening section with background image and overlay
  • <section class="stats-bar"> — Row of four key statistics about the water crisis
  • <section class="mission-wrap"> — Three content cards using the shared .card class
  • <article class="card"> — Semantic article elements for each mission card, each containing a heading, image, description, list, and button
  • <section class="gallery-wrap"> — Interactive tiles using float layout, each linking to a student content page
  • <div class="tile-info"> — Hidden overlay inside each tile that slides into view on hover

CSS Techniques

  • Position: absolute — Used on the hero background image and colour overlay so they sit behind the text content
  • Position: relative with z-index — Applied to the hero content so it appears on top of the background layers
  • Opacity — The background image uses opacity: 0.30 to keep it subtle and readable
  • Float layout — The stats bar, card grid, and tile gallery all use float: left with percentage widths
  • Hover with bottom positioning — The tile info overlay is positioned at bottom: -60px by default and moves to bottom: 0 on hover, creating the slide-up reveal effect
  • Overflow: hidden — Applied to tile containers so the info panel is hidden until hover

Design Rationale

The hero section opens with a strong emotional statement "Millions still have no access to safe, clean water" paired with a real photograph of the crisis. This was a deliberate choice to immediately connect visitors with the human reality of SDG 6 rather than leading with abstract statistics. The three mission cards each contain a list of three points as required by the brief, covering the crisis, community impact, and hopeful outcomes. The interactive gallery tiles at the bottom use a hover slide-up effect to make the page feel engaging without relying on JavaScript.

5. Gallery Page (gallery.html)

The Gallery Page presents six photographs related to the water crisis in Sub-Saharan Africa. Users can click any thumbnail to expand it into a modal view with a full description and customisation controls.

HTML Structure

  • <div class="thumb-grid"> — Float-based 3-column grid of thumbnail cards
  • <div class="thumb-card"> — Each card has a tabindex for keyboard access, a data-id attribute for JavaScript, an image, a title, and a short label
  • <div class="modal-overlay"> — The full-screen modal container, hidden by default, shown by JavaScript
  • <div class="modal-body"> — Inside the modal: the image, title, description, and the colour/font customisation controls
  • <select> — Two dropdown menus for colour scheme and font style, both JavaScript-driven

JavaScript Implementation

The gallery uses JavaScript throughout, as required by the brief. The key interactions are:

Hover Interaction

Each thumbnail card has a mouseover and mouseout event listener attached via JavaScript. When the user hovers over a card, JavaScript adds the class hovered to it. CSS then uses that class to show the expand badge and lift the card. This satisfies the brief's requirement that hover interaction logic must be handled by JavaScript event listeners, not CSS alone.

Click-to-Expand Modal

When a card is clicked, the openModal(id) function runs. It looks up the card's data-id attribute, retrieves the matching entry from the items object (which stores each image's source, alt text, title, and description), and populates the modal with that data using textContent and src assignments. The modal overlay then becomes visible by adding the open class, which switches its display from none to block. Focus is moved to the close button for accessibility.

Close Interaction

The modal can be closed in three ways, all implemented in JavaScript: clicking the close button, clicking anywhere outside the modal box (detected by checking if e.target === overlay), or pressing the Escape key (detected via a keydown event listener on the document).

Colour and Font Customisation

Inside the modal, two dropdown menus allow the user to change the colour scheme and font style of the expanded view. A colours object stores background, text, and heading colours for four themes (Default, Ocean, Midnight, Warm Sand). A fonts object stores font stacks for Lato, Serif, and Monospace. The applyStyle() function reads the current dropdown values and applies the corresponding styles directly to the modal content element. These customisations apply only to the modal content the rest of the page is unaffected.

6. Content Page (content_ST1.html)

My individual content page is titled The Scale of the Crisis. It provides an in-depth, data-driven look at the global water crisis, focusing specifically on Sub-Saharan Africa. The page is divided into six sections with internal navigation.

HTML Structure

  • <aside class="side-nav"> — A sticky side navigation panel with anchor links to each section
  • <article class="article"> — The main content area floated to the right of the sidebar
  • <section id="overview"> through <section id="action"> — Six clearly defined content sections
  • <h1> — Single page title in the banner
  • <h2> — Section headings, one per section
  • <h3> — Sub-headings within sections (e.g. Climate Change, Conflict and Governance)
  • <blockquote class="pull-quote"> — A styled quotation block
  • <div class="stat-row"> — Rows of dark stat boxes showing key numbers
  • <a href="#top" class="go-top"> — A CSS-only fixed Go to Top button

CSS Techniques

  • Float layout for two columns — The side nav floats left at 200px width, and the article uses margin-left: 240px to sit beside it
  • Position: sticky — The sidebar stays visible as the user scrolls through long sections
  • Position: fixed — The Go to Top button stays in the bottom-right corner at all times, implemented in CSS only with no JavaScript
  • Box-shadow and border-radius — Applied to images to give them a polished, professional appearance
  • Float image pairs — Two images side by side using float: left at 47% width each
  • Scroll-margin-top — Applied to each section so anchor links account for the sticky header height

Design Rationale

The content page is deliberately long and information-dense because it is meant to function as a genuine educational resource about the water crisis. The sticky sidebar navigation makes it easy to jump between sections without excessive scrolling. The dark stat boxes create visual contrast that draws the eye to the most important numbers. Images are used to break up the text and provide emotional context for the statistics.

7. Challenges & Lessons Learned

Challenge 1 — JavaScript and the Hover Interaction

One of the most difficult parts of this project was understanding that the brief required hover interaction logic to be handled by JavaScript, not CSS alone. My first instinct was to use the CSS :hover pseudo-class to show and hide elements which is simpler and works visually. However, re-reading section 4.3 of the brief made it clear that the interaction logic itself had to use JavaScript event listeners.

I resolved this by attaching mouseover and mouseout event listeners to each thumbnail card. These listeners add and remove a .hovered CSS class, which then triggers the visual change. This approach separates the logic (JavaScript) from the presentation (CSS) in the way the brief intended.

What I learned: reading the brief carefully word by word is essential. What seems like a small CSS-only solution can miss a specific requirement entirely.

Challenge 2 — The Hover Slide-Up Effect on the Home Page Tiles

On the Home Page, the content gallery tiles need to reveal additional information when the user hovers over them. I wanted the info overlay to slide up from below the image, but achieving this in a clean way without using transform: translateY() (which I wanted to keep simple) required a different approach.

The solution I used was to position the info panel at bottom: -60px by default, placing it just out of view below the tile. The tile itself has overflow: hidden, so the panel is invisible until hover. On hover, CSS changes the position to bottom: 0, and a transition on the bottom property creates the smooth slide-up effect. This is a basic CSS positioning technique but produces a polished result.

What I learned: you do not always need advanced CSS properties to create good interactive effects. Understanding how position and overflow work together can achieve the same results with simpler code.

Challenge 3 — The Gallery Modal and JavaScript Data Population

Building the gallery modal was the most complex JavaScript task I had to complete. The challenge was making one single modal work correctly for six different images each needing a different image source, alt text, title, and description without duplicating the modal six times in the HTML.

I solved this by storing all the image data in a JavaScript object called items, where each key is a card ID matching the data-id attribute on the HTML card. When a card is clicked, JavaScript reads the card's data-id, looks it up in the items object, and uses the result to populate the modal. This is a clean, scalable approach adding a seventh image only requires adding one more entry to the object.

What I learned: JavaScript objects are a very useful way to store and retrieve structured data. Understanding this pattern made the gallery much easier to build and maintain.

Challenge 4 — Colour and Font Customisation in the Modal

The brief requires that the gallery modal includes controls for changing the colour scheme and font style, and that these changes apply only to the modal content not to the rest of the page. I initially found this confusing because I was not sure how to apply styles dynamically without affecting global styles.

The solution was to apply the style changes directly to the modal content element using element.style.backgroundColor, element.style.color, and element.style.fontFamily. Because these are inline styles on a specific element, they do not affect anything outside of it. The applyStyle() function reads both dropdowns and applies the matching values from the colour and font objects in one call.

What I learned: applying styles directly through JavaScript's style property is a simple and reliable way to change the appearance of a specific element without touching the stylesheet.

8. Accessibility Considerations

Accessibility was considered throughout the development of all my pages. The following specific techniques were applied:

Semantic HTML

All pages use correct semantic HTML elements throughout. The <header>, <main>, <footer>, <nav>, <article>, <section>, and <aside> elements are used for their correct purposes. This means screen readers can identify the structure of the page and allow users to navigate between regions without relying on visual cues.

Alternative Text on All Images

Every image across all three of my pages has a descriptive alt attribute. For decorative background elements (the hero background image and overlay), role="presentation" is applied so screen readers skip them entirely. For meaningful images, the alt text describes what is shown and why it is relevant to the page content.

Keyboard Navigation

The gallery thumbnail cards are fully keyboard accessible. Each card has tabindex="0" which adds it to the tab order. Keyboard users can press Enter or Space to open the modal, and Escape to close it. Focus is moved to the close button when the modal opens so keyboard users do not lose their place.

ARIA Attributes

The navigation has aria-label="Main navigation" to identify it to screen readers. The modal uses role="dialog", aria-modal="true", aria-labelledby, and aria-hidden="true/false" which is toggled by JavaScript when the modal opens and closes. The active nav link has aria-current="page". Gallery cards have descriptive aria-label attributes explaining what they contain.

Heading Hierarchy

All pages follow a logical heading hierarchy. Each page has exactly one <h1> for the page title, followed by <h2> for main sections, and <h3> for sub-sections. This structure is important for screen reader users who navigate by heading.

Colour Contrast

The colour palette was chosen to ensure sufficient contrast between text and background throughout the site. White text on the dark navy background and dark text on the light surface background both meet readability standards. Interactive elements such as buttons and links have visible hover states to confirm interactivity.

Why These Choices Improve Usability

These accessibility features ensure the website is usable by people with visual impairments using screen readers, people who cannot use a mouse and rely on keyboard navigation, and people using assistive technologies of any kind. Accessible design also benefits all users clear structure, readable text, and logical navigation improve the experience for everyone regardless of ability.

All pages I have created comply with JANET regulations, which govern the responsible publication of web content on the UK academic network. The following explains how compliance was achieved across my work:

All written content on my pages is original, factual, and directly related to the United Nations Sustainable Development Goal 6 — Clean Water and Sanitation. No content promotes hatred, violence, discrimination, or any activity that would be considered inappropriate for an academic publication. The tone throughout is informative, serious, and respectful of the communities depicted.

Responsible Use of External Resources

All images used on my pages are either provided by the team from appropriate sources or used with proper attribution. External resources, including the Google Fonts API for the Lato typeface, are referenced correctly using standard link elements. No copyrighted material has been reproduced without proper acknowledgement. All image alt attributes include clear descriptions that also serve as informal attribution of the depicted subjects and regions.

No Unlawful Activity

None of the pages I created contain links to, or promotion of, any unlawful activity. All external links point to legitimate, publicly available web resources. The website does not collect personal data, and no server-side processing is involved, all interaction is handled client-side in the browser.

Academic Integrity

All code in my pages was written by me. I have not copied code from other students or from pre-built website templates. Where techniques were learned from lecture notes or online tutorials, these are referenced in the References section of this Page Editor. The content of the website is original and was not reproduced from the United Nations SDG website or any other source.

Professional Conduct

The website presents a professional and appropriate image consistent with the University of Westminster's expectations. The subject matter global water crisisis treated with the seriousness and sensitivity it deserves. No content could be considered offensive, misleading, or inappropriate for an academic audience.
