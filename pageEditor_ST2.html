2. Splash Screen – Entry Portal (splash.html)

↗ View Page

Purpose & Branding

The splash screen serves as the professional entry point for ProjectBlue. It reinforces the branding of UN SDG Goal 6 and provides the user with an immersive introduction before redirecting to the main content.

Technical Implementation

  • Auto-Redirect: Uses <meta http-equiv="refresh" content="4;url=home.html"> to automatically navigate to the home page after 4 seconds.
  • CSS Loading Animation: A custom-built .loader-bar using CSS @keyframes synchronized with the 4-second redirect timer. This replaces the need for a heavy GIF.
  • Typography & Palette: Utilizes the Lato font family and the ProjectBlue dark navy/accent blue color scheme for visual consistency.
  • JavaScript Countdown: A setInterval function updates the #timer element in real-time, providing visual feedback to the user.
  • Skip Functionality: A "Skip Intro" button allows users to bypass the animation immediately, improving user experience for returning visitors.

Accessibility

The splash screen includes aria-live="polite" on the countdown timer so screen readers announce the remaining time. Descriptive aria-label attributes are used on the skip button and the SDG badge to ensure the page's purpose is clear to all users.

3. Content Page – Dirty Water, Hidden Danger (content_ST2.html)

↗ View Page ↗ Validation Evidence

HTML Structure

The page uses the shared HTML template structure provided by Student 1, with a <header>, <main>, and <footer>. Inside <main>, the content is organised into distinct semantic regions:

  • A full-width dark editorial hero banner (.content-hero) with breadcrumb, page title, byline, and introduction paragraph
  • A sticky internal navigation bar (<nav class="content-nav">) with anchor links to each of the five content sections
  • A white content body area (.content-body) containing five <section> elements, each with a unique id attribute matching the internal nav anchors

The five content sections are: What Is Contaminated Water?, Health Effects, Key Waterborne Diseases, Who Is Most Affected?, and What You Can Do. Each section uses a consistent structure of <h2> headings, <h3> subheadings, body paragraphs, and supporting visual elements.

  • One <h1> used in the hero banner for the page title — no other <h1> appears on the page
  • <h2> headings mark each of the five major sections
  • <h3> headings introduce subsections (e.g. "Acute Health Effects", "Chronic Health Effects")
  • All four images include descriptive alt attributes and are followed by <p class="img-caption"> captions
  • A two-column image layout uses a <div class="img-pair"> wrapper with float-based CSS layout
  • Three stat callout boxes (<div class="stat-callout">) highlight key statistics with large numerals
  • The waterborne diseases section uses a fully structured <table> with <thead>, <tbody>, and scope="col" on all header cells
  • A fixed "Go to Top" button uses href="#top" pointing to a named anchor at the document start

CSS Techniques

All page-specific styles are contained within an embedded <style> block in the <head> of content_ST2.html, as required by the coursework specification. The global style.css file provides the shared header, footer, typography base, and CSS custom properties (variables). The embedded CSS adds the following page-specific styles:

  • Content hero (.content-hero): Dark navy background (#0a1628) with padding, a bottom border accent, breadcrumb styling, large bold h1 with an <em> accent colour, and a semi-transparent intro paragraph.
  • Sticky internal navigation (.content-nav): Uses position: sticky; top: 0; with z-index: 50 to remain fixed at the top of the viewport during scroll. A flex layout with flex-wrap: wrap allows nav items to reflow on smaller screens. On mobile, flex-wrap: nowrap and overflow-x: auto create a horizontally scrollable strip.
  • Content sections (.content-section): Centred with max-width: 860px and margin: 0 auto. Sections are separated by a bottom border (1px solid #e8f0f7), removed on the last section using :last-of-type.
  • Images and captions: .content-img is full-width with a light border and box shadow. .img-caption uses italic styling, small font size, and centred alignment. The .img-pair two-column layout uses float: left with calc(50% - 10px) width for each image, and overflow: hidden on the wrapper to contain floats.
  • Stat callout boxes (.stat-callout): Dark navy background with a 4px left border in the accent colour (var(--accent-light)). The large statistic figure (.big-stat) uses a 3rem font size in the accent blue.
  • Disease table (.disease-table): Full-width with collapsed borders. Header row uses dark navy background with accent-coloured text and uppercase letter-spacing. Alternating row background applied using tr:nth-child(even) td.
  • Go to Top button (.go-top): Fixed position at bottom-right using position: fixed; bottom: 32px; right: 32px; with a hover colour transition.
  • Responsive breakpoint: A single @media (max-width: 680px) rule reduces the hero h1 font size, stacks image pairs vertically, and makes the content nav horizontally scrollable.

Content Summary

The page covers five thematic sections focused on SDG 6 – Clean Water and Sanitation. The content investigates the sources and types of water contamination, the acute and chronic health effects of unsafe water, a structured overview of key waterborne diseases (cholera, typhoid, hepatitis A, arsenicosis, schistosomiasis), the populations most affected (children under five, women and girls, conflict-affected communities), and concrete actions individuals can take. All content is original, written specifically for this project. Statistics are sourced from WHO and UNICEF publications and are cited in the References section. A link to the Action Impact Simulator (ais.html) is provided at the end of the final section.

Design Rationale

The dark editorial hero was chosen to create a strong visual contrast with the clean white content body below, reinforcing the serious, documentary tone of the subject matter. The sticky internal navigation bar allows users to jump between sections on a long-form page without losing their position. Stat callout boxes break up prose-heavy sections and draw attention to the most impactful data points. The float-based image pair layout was kept consistent with the shared style.css conventions used across the site, avoiding the need for a separate flex or grid context that might conflict with global styles.

4. Accessibility Considerations

Accessibility was considered throughout the design and implementation of content_ST2.html. The following techniques were applied to meet WCAG 2.1 Level AA requirements:

Semantic HTML

All structural elements use semantic HTML5: <header>, <nav>, <main>, <section>, and <footer>. This provides meaningful document structure for screen readers and assistive technologies without relying on visual presentation alone.

Heading Hierarchy

A single <h1> appears in the hero banner. All five content sections use <h2>, and subsections within those sections use <h3>. No heading levels are skipped, ensuring a logical outline for screen reader navigation.

Alternative Text

Every <img> element includes a descriptive alt attribute that explains both the content and the context of the image — not just a filename. For example, the image pair includes distinct alt text for each: the cholera microscope image and the child drinking from an unsafe source.

ARIA Attributes

  • aria-label used on both <nav> elements to distinguish the main site navigation from the internal page section navigation
  • aria-label applied to all five <section> elements describing their content
  • aria-label="Key statistic" on each stat callout box so screen readers announce its purpose
  • aria-label on the waterborne disease <table> providing a full description of its contents
  • aria-label="Go back to the top of the page" on the fixed "Go to Top" anchor button
  • aria-hidden="true" on the decorative top anchor (<a id="top">) to prevent it being announced as a link
  • aria-label on the image pair <div> to provide context for the two images

Table Accessibility

The disease data table uses <thead> and <tbody> to separate header and data rows. All four column headers use scope="col" to associate them correctly with their respective columns for screen reader users. The table carries an aria-label attribute describing its full purpose.

Colour and Contrast

The white text and accent blue (#5ec4f0) used on the dark navy backgrounds (#0a1628, #0c1c2e) meet the WCAG AA contrast ratio requirement of 4.5:1 for normal text. Body text (#3a5166) on the white content area also meets the minimum contrast threshold.

Keyboard Navigation

All interactive elements — links, the internal nav, and the "Go to Top" button — are native HTML anchor elements and are therefore natively keyboard focusable. The internal navigation allows keyboard users to skip directly to any section. The sticky nav remains accessible at all times during scroll.

6. Challenges & Lessons Learned

Challenge 1: Sticky Navigation Bar Conflicting with Scroll Position

The internal navigation bar uses position: sticky; top: 0; to remain visible as the user scrolls. An initial problem arose where the sticky nav would overlap section headings when an anchor link was clicked — the browser would scroll the target section to the very top of the viewport, placing the heading behind the nav bar. This was resolved by adjusting the z-index of the nav (z-index: 50) and adding position: -webkit-sticky; for Safari compatibility, ensuring the nav displays correctly across all tested browsers.

Challenge 2: Float Clearance in the Image Pair Layout

The two-column .img-pair layout uses CSS floats (float: left) to place two images side by side. Without a clearfix solution, the parent <div> would collapse to zero height because floated elements are removed from normal document flow. The fix was to add overflow: hidden to the .img-pair container, which forces the parent to expand to contain its floated children. This is a well-established clearfix technique that avoids adding extra markup.

Challenge 3: Keeping Embedded CSS Scoped to This Page Only

Because content_ST2.html embeds a <style> block that targets elements like h2 and p, there was a risk of those rules conflicting with the global styles from style.css if selectors were not specific enough. This was managed by always scoping embedded rules to a parent class — for example, .content-section h2 rather than a bare h2 — so that the embedded styles only affect elements within the content body and do not bleed into the shared header or footer.

Lessons Learned

  • Defining section IDs and anchor links before writing any CSS made the internal navigation trivial to implement and test.
  • Testing the page at mobile widths early (using Chrome DevTools) revealed the image pair overflow issue before it became a larger layout problem.
  • Writing comments in the embedded CSS to label each numbered section (matching the specification's required CSS technique list) made the final review and validation process significantly faster.
  • Using CSS custom properties (var(--accent-light)) from the global stylesheet throughout the embedded styles kept colour consistency without hardcoding values in multiple places.
